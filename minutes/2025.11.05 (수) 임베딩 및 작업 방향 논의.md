### 📎 안건 1: 고차원 데이터셋 처리 방법 Q&A

**Q1. 임베딩 생성이 불가능한 고차원 데이터셋 처리 방법?**
- 임베딩 전에 차원 축소(PCA, random projection 등) 적용 추천
- 963차원에서 32~64차원 정도로 줄이는 것은 충분히 합리적
- 단, 차원 축소 시 시간적 구조와 상관관계를 유지해야 함

**Q2. VRAM이 허용하는 범위로 차원 축소 필요?**
- 차원 축소를 적용한다면, 모든 데이터셋을 동일한 입력 차원으로 맞춰야 함
- 이미 생성해둔 4종 데이터의 임베딩도 다시 추출 필요
- 입력 차원이 다르면 모델이 일반화되지 않기 때문

**Q3. Dimension이 작은 경우 축소하는 차원만큼 Padding을 넣어도 되는지?**
- 작은 차원의 데이터셋은 패딩(예: 0 또는 학습된 패딩 토큰)을 이용해 통일된 차원(예: 32차원)으로 맞추면 됨
- 단, 패딩 값이 노이즈로 작용하지 않도록 주의 필요
- 모델이 이를 중립적으로 처리하도록 설계 필요

**Q4. 여러 도메인을 하나의 임베딩 벡터로 만들기와 하나의 임베딩 벡터로 여러 다운스트림 태스크를 진행하는 것이 둘 다 동시에 가능한가?**
- 종프 기간 내에는 불가능
- 하나의 임베딩 벡터로 여러 다운스트림 태스크를 진행해야 함

# 프롬프트
> 교차학습이 뭔지 알려줘

교차학습(인터리빙, interleaving)은 한 번의 공부 세션 안에서 서로 다른 주제나 문제 유형을 번갈아 섞어서 학습하는 방법이다. 예를 들어 수학을 공부할 때 덧셈만 쭉 푸는 대신, 덧셈·뺄셈·곱셈·나눗셈 문제를 섞어서 푸는 식이다.​​

왜 효과적인가
같은 유형만 반복하는 블록식 학습보다, 장기 기억 유지와 이해도, 전이(새 문제에 응용) 능력이 더 좋아지는 경향이 있다.​

서로 다른 유형을 번갈아 풀면서 “이 문제에는 어떤 개념·공식이 맞는가?”를 계속 구분해야 해서, 개념 변별 능력과 인출 연습이 강화된다.​

블록식 학습과의 차이
블록식 학습: 한 단원(예: 미분)만 몰아서 푼 뒤, 다음 단원으로 넘어가는 방식으로, 단기 성취감은 높지만 시간이 지나면 성적 유지가 떨어지기 쉽다.​

교차학습: 여러 단원·유형을 섞어 풀기 때문에 연습 중 체감 난이도와 오답률은 더 높게 느껴지지만, 일정 시간이 지난 뒤 테스트에서는 더 좋은 결과를 보이는 사례가 반복 보고된다.​​

어떻게 적용하나
관련은 있지만 다른 유형을 섞는다: 예를 들어 수학에서 방정식·도형·확률 문제를 한 세션에 섞거나, 프로그래밍에서 알고리즘 문제 유형을 2~3가지 섞는다.​

너무 자주 바꾸지는 않는다: 몇 분마다 갈아치우기보다, 한 유형을 적당히 몰입해서 풀다가 “자동 모드”에 들어간 느낌이 들 즈음 다른 유형으로 바꾸는 것이 좋다고 권장된다.​

교차학습이 특히 잘 맞는 경우
서로 비교·대조가 가능한 개념이 많은 과목(수학, 물리, 프로그래밍 문제 풀이, 언어 문법/독해 등)에서 효과가 크다고 보고된다.​

이미 기초를 어느 정도 익힌 학습자가, 개념을 더 깊이 이해하고 시험에서 섞여 나오는 문제에 대비할 때 적합하다.​​

간단한 실전 예시 (개발자 기준)
알고리즘 공부 시:

30분: 그래프 탐색 문제

30분: DP 문제

30분: 그리디/정렬 응용 문제
이런 식으로 하루 세션 안에 유형 2~3개를 돌리면서, 전날·전전날 풀었던 유형도 섞어 복습하면 교차학습과 분산학습 효과를 함께 얻을 수 있다.​
